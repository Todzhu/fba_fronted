# 单细胞流程详情页紧凑工作台布局设计

## 背景

单细胞分析流程详情页当前存在两个主要布局问题：

- 顶部“分析步骤”区域使用大卡片展示 6 个步骤，占据首屏过多高度。
- 右侧“结果预览”按所有指标、图表、表格顺序完整展开，图表较多时会把页面拉得很长，参数区和结果区的浏览节奏被打断。

本次优化目标是把详情页从“长报告页面”调整为“分析工作台”：用户在首屏即可看到当前步骤、核心参数、关键结果和下一步操作。

## 目标

- 压缩顶部步骤导航高度，让它只承担流程定位和步骤切换职责。
- 压缩当前步骤标题区，减少重复大图标和大留白。
- 结果预览改为固定高度的右侧面板，内部滚动，不再无限撑高页面。
- 每个步骤默认展示最关键的指标和图表，更多内容通过“查看全部”展开。
- 保持现有流程、参数、运行、日志、保存 h5ad 等业务逻辑不变。

## 非目标

- 不重做后端流程执行逻辑。
- 不改变步骤数量、步骤顺序和状态含义。
- 不移除已有结果文件，只调整默认预览数量和展示方式。
- 不在本次设计中重做完整结果页 `PipelineResult.vue`。

## 页面结构

新的页面结构从上到下为：

1. 项目头
2. 紧凑流程条
3. 当前步骤工具栏
4. 主工作区

### 项目头

项目头保留返回按钮、项目名称、项目描述。减少上下 padding，使其只作为当前位置提示。

项目头内容：

- 返回按钮
- 项目名
- 项目描述
- 流程摘要，例如 `5/6 已完成`

### 紧凑流程条

使用紧凑横向步骤导航替代当前大卡片步骤区。

每个步骤展示：

- 状态图标和序号
- 步骤名称
- 当前步骤高亮边框或背景

状态规则：

- 已完成：绿色对勾
- 当前步骤：青绿色描边
- 运行中：蓝色动态图标或状态点
- 待运行：灰色序号
- 失败：红色警示图标

高度目标：`72px - 88px`。

交互规则：

- 点击步骤切换 `activeStepIndex`。
- 保留状态可见性。
- 移除右侧状态图例，因为图标和颜色已经表达状态。

### 当前步骤工具栏

当前步骤标题区域改为一条紧凑工具栏。

左侧展示：

- `步骤 N · 步骤名称`
- 一行上下文信息，例如数据路径、组织类型、目标细胞类型等。

右侧展示：

- 使用说明
- 日志
- 开始运行 / 重新运行
- 细胞注释完成后的保存 h5ad

高度目标：`72px - 88px`。

## 主工作区

桌面端使用左右两栏：

- 左侧：参数、样本表、Marker 表、Cluster 注释表等主要操作区。
- 右侧：结果预览面板。

桌面端布局：

```text
grid-template-columns: minmax(0, 1fr) minmax(420px, 520px)
gap: 20px
align-items: start
```

右侧结果面板使用 sticky 布局：

- `position: sticky`
- 顶部偏移匹配全局导航和紧凑工具栏。
- 最大高度：`calc(100vh - 180px)`。
- 面板内部滚动。

移动端改为单列：

- 当前步骤参数在上。
- 结果预览在下。
- 图表卡片单列显示。

## 结果预览设计

结果预览面板分为四个区域：

1. 状态区
2. 指标区
3. 图表区
4. 表格区

### 状态区

状态区固定高度，文案简短：

- 待运行：`等待运行`
- 运行中：`正在运行`
- 已完成：`当前步骤已完成`
- 失败：`运行失败，请查看日志`

### 指标区

默认显示最多 4 个关键指标。超过 4 个时显示“更多指标”入口。

指标字段需要尽量中文化，不直接暴露后端字段名：

- `n_samples` → `样本数`
- `n_groups` → `分组数`
- `total_cells` → `细胞数`
- `total_genes` → `基因数`
- `cells_after_filter` → `过滤后细胞`
- `filter_ratio` → `过滤比例`
- `genes_after_filter` → `过滤后基因`
- `median_genes` → `中位基因数`
- `n_clusters` / `clusters` → `Cluster 数`
- `n_cell_types` → `细胞类型数`
- `organism` → `物种`
- `database` → `数据库`

### 图表区

默认只显示前 4 张图表。

图表卡片规则：

- 固定比例容器，使用 `4:3`。
- 图片使用 `object-contain`。
- 文件名单行截断。
- 点击图片打开现有 lightbox。
- 图片加载失败时显示明确 fallback。

超过 4 张图时显示：

- `查看全部图表（N）`

“查看全部图表”打开右侧抽屉，展示全部图表。该抽屉使用独立滚动，不影响主页面高度。

### 表格区

表格和图表分开展示。

默认显示最多 3 个表格文件，超过时显示：

- `查看全部表格（N）`

每个表格项显示：

- 文件名
- 文件图标
- 点击打开或下载

## 各步骤默认展示重点

### 数据读取

重点指标：

- 样本数
- 分组数
- 细胞数
- 基因数

默认结果重点是样本表，图表不是主入口。

### 质控过滤

重点指标：

- 过滤后细胞
- 过滤比例
- 过滤后基因
- 中位基因数

默认图表：

- 总体 violin
- 分样本 violin
- 其余图表进入“查看全部图表”

### 降维聚类

重点指标：

- Cluster 数
- 细胞数
- 分辨率

默认图表：

- UMAP cluster
- UMAP group
- tSNE 或 PCA 图表进入更多图表

### 特征基因

重点指标：

- Cluster 数
- Marker 数

默认图表：

- top marker dotplot
- heatmap

表格入口需要突出，因为 marker 表是该步骤的重要产物。

### 细胞注释

重点指标：

- 细胞类型数
- 物种
- 数据库或注释方法

默认图表：

- 注释 UMAP
- 细胞比例图
- 用户生成的 Dotplot / Feature Plot

保存 h5ad 按钮放在当前步骤工具栏右侧。

### 亚群分析

待运行时显示简短引导。

完成后展示：

- 目标细胞类型
- 亚群数量
- 亚群 UMAP
- 亚群 marker 结果

## 组件调整范围

### `PipelineStepper.vue`

职责调整为紧凑流程条：

- 删除大卡片布局。
- 保留步骤点击切换。
- 保留状态表达。
- 控制高度和横向溢出。

### `PipelineDetail.vue`

调整页面结构：

- 压缩项目头。
- 将当前步骤标题改为工具栏。
- 调整 `step-workspace-grid` 两栏比例。
- 让结果面板 sticky。
- 保持注释步骤 Marker 面板和 Cluster 注释表逻辑不变。

### `StepResultPanel.vue`

职责调整为“关键结果预览”：

- 面板有限高，内部滚动。
- 指标默认最多 4 个。
- 图表默认最多 4 张。
- 表格默认最多 3 个。
- 提供“查看全部图表”和“查看全部表格”入口。

## 错误和空状态

- 图片加载失败：显示“图片加载失败”和文件名。
- 没有图表和表格：显示“当前步骤没有可预览的图表或表格”。
- 待运行：显示运行前引导，不展示空白结果容器。
- 运行中：显示加载状态和日志入口。
- 失败：显示失败状态，并引导用户查看日志。

## 测试策略

新增或更新源码测试，覆盖：

- 步骤导航使用紧凑布局，不再依赖大卡片高度。
- 结果面板具备最大高度和内部滚动。
- 图表默认截断展示，存在查看全部入口。
- 表格默认截断展示，存在查看全部入口。
- 注释步骤的保存 h5ad 按钮仍保留。
- 日志按钮仍在当前步骤工具栏中。

## 验收标准

- 顶部步骤区域高度明显低于当前大卡片布局。
- 首屏可以同时看到当前步骤工具栏、左侧主要参数区和右侧关键结果。
- 图表很多时，页面不会被结果预览无限拉长。
- 点击图片仍可放大预览。
- 用户仍可切换步骤、查看日志、运行或重新运行步骤。
- 细胞注释步骤仍可保存 h5ad。
